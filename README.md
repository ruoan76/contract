# 合同审批管理平台 - 项目总览

## 📋 项目简介

全生命周期数字化合同审批管理平台，实现合同起草、线上流转、多级审批、法务审核、用印管控、归档台账、风险预警、数据溯源一体化管控。

## 🎯 核心目标

| 目标 | 说明 |
|------|------|
| **规范审批流程** | 标准化合同起草→审批→签署→归档全流程 |
| **严控法律风险** | AI+人工双重审查，识别合同风险点 |
| **严控经营风险** | 财务条款、履约能力、信用风险评估 |
| **提升审批效率** | 自动化流转、智能分派、并行审批 |
| **数据可查可控** | 全链路数据溯源、操作审计、统计分析 |

## 🔑 核心模块

### 1. AI 合同审查（重点）

| 审查维度 | 说明 | 技术实现 |
|---------|------|---------|
| **合规性审查** | 法律法规符合性、强制性条款 | Qwen3.6 + 法规知识库 |
| **风险条款识别** | 违约责任、赔偿上限、管辖约定 | 语义分析 + 规则引擎 |
| **财务审查** | 付款条件、价格条款、税务条款 | NER 实体识别 + 规则校验 |
| **履约能力评估** | 交付周期、质量标准、验收条件 | 文本分类 + 相似度匹配 |
| **异常检测** | 非常规条款、隐藏风险、矛盾表述 | 异常检测 + 对比分析 |

### 2. 审批流程

- **多级审批**：部门主管→法务→财务→高管
- **条件分支**：根据合同金额/类型自动路由
- **并行审批**：多部门同时审核
- **会签机制**：多人会签、少数服从多数
- **退回重审**：审批不通过退回起草人

### 3. 用印管控

- **电子印章**：CA 认证电子印章
- **用印审批**：用印前二次确认
- **用印记录**：谁在何时用印何文件
- **防伪追溯**：水印 + 哈希校验

### 4. 风险预警

| 预警类型 | 触发条件 | 响应方式 |
|---------|---------|---------|
| **高风险条款** | AI 识别重大风险 | 自动拦截 + 法务介入 |
| **履约异常** | 交付逾期、质量不达标 | 邮件/企微通知 |
| **到期提醒** | 合同到期前 30/15/7 天 | 自动提醒续签 |
| **信用风险** | 对方信用评级下降 | 预警 + 审批升级 |

## 🚀 开发计划

### V1 MVP（核心主线，8 周）

| 阶段 | 内容 | 周期 | 交付物 |
|------|------|------|--------|
| **Sprint 1** | 基础架构 + 用户权限 + 相对方管理 | 2 周 | 用户系统、相对方档案、黑名单 |
| **Sprint 2** | 合同 CRUD + 版本管理 + 文件上传 | 2 周 | 合同起草、模板引用、草稿保存 |
| **Sprint 3** | 模板管理（5 种标准模板） | 1 周 | 模板库、版本管理、发布审批 |
| **Sprint 4** | AI 审查（合规 + 风险 + 置信度 + **知识库基座**） | 2 周 | AI 审查引擎、规则引擎、审查报告、50 条法规库 |
| **Sprint 5** | 审批流程（3 种标准流程 + **配置化**） | 1 周 | 流程引擎、待办列表、审批操作、金额/审批人配置 |
| **Sprint 6** | 用印归档 + 台账 + **状态看板** + 移动端通知 | 1 周 | 用印记录、台账查询、到期提醒、飞书/企微通知 |

## 📊 数据库设计（概要）

| 表名 | 说明 | 核心字段 |
|------|------|---------|
| `contracts` | 合同主表 | id, title, type, status, amount, parties |
| `contract_versions` | 合同版本 | id, contract_id, version, content, created_at |
| `approval_flows` | 审批流程 | id, contract_id, current_step, status |
| `approval_steps` | 审批节点 | id, flow_id, step, approver, action, comment |
| `ai_reviews` | AI 审查记录 | id, contract_id, version, result, risk_level |
| `risk_alerts` | 风险预警 | id, contract_id, type, level, message |
| `audit_logs` | 审计日志 | id, user, action, resource, timestamp |
| `seal_records` | 用印记录 | id, contract_id, user, timestamp, hash |

## 🔐 安全设计

- **数据加密**：敏感字段 AES-256 加密
- **传输安全**：HTTPS + JWT Token
- **权限控制**：RBAC 角色权限
- **操作审计**：全链路操作记录
- **数据备份**：每日自动备份 + 异地容灾
