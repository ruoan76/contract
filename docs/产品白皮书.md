## 1 产品概述

### 1.1 产品简介

合同审批管理平台是一款面向企业级用户的全生命周期数字化合同管理解决方案。平台覆盖"起草→审批→签署→履约→归档"全流程，以 **AI 智能审查**为核心能力，融合规则引擎与大语言模型（Qwen3.6），实现合同风险的自动识别、量化评估与智能预警，帮助企业降低法律合规风险、提升审批效率、实现数据可查可控。

### 1.2 产品定位

| 维度 | 定位 |
|------|------|
| **目标用户** | 中大型企业法务、财务、业务、管理层 |
| **核心场景** | 合同全生命周期管理 + AI 智能风险审查 |
| **差异化能力** | RAG 增强 + 规则引擎 + 置信度评估 + 策略驱动配置 |
| **部署方式** | 本地化部署 / Docker 一键部署，支持离线 AI 推理 |

### 1.3 核心能力一览

| 能力 | 描述 |
|------|------|
| **智能起草** | 模板驱动 + 相对方档案自动带入 + 版本管理 |
| **AI 审查** | 五维度风险审查（合规/风险/财务/履约/异常），30 秒内输出结构化报告 |
| **流程审批** | 3 种标准审批流程，按金额/类型自动路由，支持条件分支与会签 |
| **用印管控** | 二次确认 + 文件哈希校验 + 用印溯源 |
| **风险预警** | 到期提醒 + 履约异常 + 信用评级下降自动预警 |
| **归档台账** | 全量合同台账查询 + Excel 导出 + 多维度统计分析 |
| **操作审计** | 全链路操作记录，不可篡改 |

## 2 行业背景与痛点

### 2.1 传统合同管理的核心痛点

| 痛点 | 表现 | 影响 |
|------|------|------|
| **流程不规范** | 线下审批口头沟通，流程依赖个人经验 | 审批周期长（平均 5 天），责任不清 |
| **法律风险失控** | 缺少专业审核，风险条款漏判率高 | 违约损失、合规处罚 |
| **效率低下** | 反复修改、版本混乱、查找困难 | 业务推进缓慢 |
| **数据不可查** | 合同台账手工维护，统计分析困难 | 管理层决策缺乏数据支撑 |
| **履约失控** | 签署后无人跟踪，到期遗忘 | 续约延误、违约风险 |

### 2.2 AI 时代的合同管理新范式

传统合同管理依赖人工法务逐条审查，面对量大面广的合同场景，人力投入与质量保障难以兼顾。本平台引入大语言模型 + 检索增强生成（RAG）+ 规则引擎三重能力，实现：

- **漏判率 <10%**（基于 50 份历史合同测试验证）
- **审查耗时 <30 秒**（10 页以内合同）
- **每条审查结论附带法律依据**（引用《民法典》等法规条文）
- **置信度分级**（高/中/低）+ 法务反馈闭环

## 5 核心功能模块

### 5.1 合同起草

合同起草是业务流程的起点，支持多种起草方式，确保合同信息标准化、结构化。

| 功能 | 说明 |
|------|------|
| **空白起草** | 手动填写合同基本信息 + 上传附件（PDF/Word/图片，最大 50MB） |
| **模板起草** | 从标准模板库选择模板，自动填充条款，3 步完成起草 |
| **历史引用** | 引用已归档合同作为新合同基础 |
| **草稿保存** | 每 5 分钟自动保存，刷新不丢失 |
| **版本管理** | 每次提交生成新版本，保留修改历史 |
| **相对方选择** | 从相对方库搜索（名称模糊搜索 / 信用代码精确搜索）或新建 |
| **文件上传** | 拖拽上传 + 进度条显示 |

### 5.2 合同模板管理

标准化模板是降低合同风险的第一道防线。

| 功能 | 说明 |
|------|------|
| **模板创建** | 上传标准合同，标记可变字段（{金额}、{日期}、{相对方}） |
| **模板分类** | 按合同类型分类采购/销售/劳务/保密/NDA，首期支持 5 种 |
| **版本管理** | 模板版本迭代，旧版本标记"已废止"，不影响已引用合同 |
| **发布审批** | 新模板/模板修订需法务负责人审批后生效 |
| **模板偏离检测** | 偏离模板的条款自动高亮（V1 示意，V2 完整） |

### 5.3 相对方管理

相对方档案是合同风险管理的基石。

| 功能 | 说明 |
|------|------|
| **相对方档案** | 名称、信用代码、法人、联系人、地址、行业分类 |
| **黑名单管理** | 标记高风险相对方，起草时自动拦截，强制法务复核 |
| **信用校验** | 信用代码 18 位格式 + 校验位自动验证 |
| **批量导入** | CSV 批量导入历史相对方数据 |
| **历史统计** | 历史合同数、总金额、违约次数、平均账期（V2） |

### 5.4 风险预警

| 预警类型 | 触发条件 | 响应方式 |
|----------|---------|---------|
| **高风险条款** | AI 识别重大风险 | 自动拦截 + 法务介入 |
| **履约异常** | 交付逾期、质量不达标 | 邮件/企微通知 |
| **到期提醒** | 合同到期前 30/15/7 天 | 自动提醒续签，推送至起草人 + 部门主管 |
| **信用风险** | 对方信用评级下降 | 预警 + 审批升级 |

### 5.5 用印管控

| 功能 | 说明 |
|------|------|
| **用印申请** | 审批通过后申请用印，选择印章类型 |
| **用印审批** | 档案管理员二次确认，未通过合同不可用印 |
| **用印记录** | 记录谁、何时、用印何文件、文件哈希 |
| **线下盖章** | 上传盖章后扫描件，人工核验 |
| **防伪追溯** | 水印 + SHA-256 哈希校验，文件不可篡改 |

### 5.6 归档与台账

| 功能 | 说明 |
|------|------|
| **合同归档** | 签署完成后一键归档，归档后不可编辑 |
| **台账查询** | 按类型/状态/相对方/金额/日期范围多维筛选 |
| **台账导出** | 导出 Excel/CSV |
| **统计报表** | 合同数量/金额/类型/风险等级统计，按月/季度/年维度 |

## 6 AI 智能审查引擎

AI 合同审查是本产品的核心差异化能力，也是行业首创的"Skill 编排 + RAG 增强 + 策略驱动"三位一体架构。

### 6.1 审查维度

| 审查维度 | 权重 | 检查项 | 评分标准 |
|----------|------|--------|---------|
| **合规性审查** | 30% | 主体资格、经营范围、强制性规定、格式条款 | 违反强制性规定 → 直接高风险 |
| **风险条款识别** | 25% | 违约责任、管辖约定、解除条件、保密条款 | 赔偿无上限 → 高风险 |
| **财务审查** | 20% | 付款条件、价、税务 | 预付款 >50% → 高风险 |
| **履约能力评估** | 15% | 交付周期、质量标准、验收条件 | 验收条件模糊 → 中风险 |
| **异常检测** | 10% | 矛盾条款、隐藏风险、缺失条款 | 条款矛盾 → 高风险 |

### 6.2 AI 审查技术架构

```
合同文档 → 文本提取 → 分段解析 → 策略匹配 → Skill 编排 → 报告
    ↑           ↑           ↑          ↑         ↑
  PDF/Word/   条款拆分    合同类型    Skill 调度
  图片 OCR
                              ↓
                       ┌──────────────┐
                       │ 五维 Skill   │
                       ├──────────────┤
                       │ 合规 Skill   │
                       │ 风险 Skill   │
                       │ 财务 Skill   │
                       │ 履约 Skill   │
                       │ 异常 Skill   │
                       └──────────────┘
                              ↓
                    ┌─────────────────────┐
                    │ 底层能力            │
                    ├─────────────────────┤
                    │ Qwen3.6 (LLM)      │
                    │ Chroma (RAG检索)   │
                    │ RuleEngine (规则)  │
                    │ CoT (推理链)       │
                    │ Self-Correction    │
                    └─────────────────────┘
```

### 6.3 核心技术能力

| 技术 | 用途 | 效果 |
|------|------|------|
| **RAG（检索增强生成）** | 结合向量索与规则引擎，解决大模型幻觉 | 审查结论附带具体法律依据 |
| **CoT（链式思维）** | 强制模型展示思考过程 | 提升审查可解释性与准确率 |
| **Few-Shot Learning** | 提供正反例，让模型学习审查标准 | 审查标准一致性提升 |
| **Structured Output** | 强制模型输出 JSON Schema | 结构化报告，下游可解析 |
| **Self-Correction（自我反思）** | 模型生成后二次校验 | 降低误报率 |
| **规则引擎** | 硬规则优先（违约金上限、管辖法院、预付款比例等） | 快速拦截已知风险 |

### 6.4 置信度分级与处理策略

| 置信度 | 处理策略 |
|--------|---------|
| **≥80%** | AI 直接标记风险等级，法务快速复核 |
| **60-79%** | AI 标记"疑似风险"，强制人工复核 |
| **<60%** | AI 标记"不确定"，转人工审查 + 记录争议点 |

### 6.5 法规知识库

内置 50 条核心法规条款，作为 AI 审查的"事实依据"。

| 法规名称 | 典型条款 | 覆盖场景 |
|----------|---------|---------|
| 《民法典》 | 第 496/584/585/686/705 条 | 格式条款、违约赔偿、违约金、保证担保 |
| 《公司法》 | 第 16 条 | 对外担保 |
| 《电子签名法》 | 第 13 条 | 电子签章 |
| 《劳动合同法》 | 第 19 条 | 试用期 |
| 《招标投标法》 | 第 46 条 | 招投标 |
| 《数据安全法》 | 第 21 条 | 数据合规 |

### 6.6 审查报告输出

AI 审查输出结构化报告，包含：

- **整体风险评级**：高/中/低 + 综合评分（0-100）
- **五维度雷达图**：合规性/风险条款/财务/履约/异常各维度评分
- **条款级审查明细**：每个条款的风险等级、问题描述、法律依据、修改建议
- **规则违规项**：硬规则检查命中项
- **修改建议汇总**：所有审查维度的可操作修改建议清单
- **导出能力**：支持 PDF/Word 格式导出

### 6.7 策略驱动配置

通过 YAML/JSON 策略文件，支持按合同类型、行业、风险偏好定制审查规则。

| 可配置项 | 说明 |
|----------|------|
| **风险偏好** | 保守/适中/激进，影响阈值和审查严格度 |
| **启用/禁用技能** | 按需加载审查维度 |
| **硬规则阈值** | 如违约金上限从 30% 调整为 25% |
| **Soft Rule Prompt** | 编辑 LLM 审查提示词模板 |
| **合同类型策略** | 采购/销售/劳务各自独立策略文件 |

### 6.8 法务反馈闭环

| 操作 | 说明 |
|------|------|
| **确认** | 法务确认 AI 审查结论 |
| **修正** | 法务修正 AI 结论，记录修正原因 |
| **误报标记** | 标记"AI 误报"，用于模型优化 |
| **漏报标记** | 标记"AI 漏报"，补充规则库 |

## 7 审批流程引擎

### 7.1 标准审批流程

| 流程名称 | 适用场景 | 审批节点 |
|----------|---------|---------|
| **标准流程** | 一般金额合同 | 起草→部门主管→法务→财务→高管→用印→归档 |
| **简易流程** | 小额合同（采购≤10万、销售≤50万） | 起草→部门主管→法务→用印 |
| **特殊流程** | 重大合同（采购>100万） | 起草→部门主管→法务→财务→高管→董事会→用印 |

### 7.2 流程特性

| 特性 | 说明 |
|------|------|
| **条件分支** | 根据合同金额/类型自动选择审批流程 |
| **并行审批** | 多部门同时审核（V2 完整） |
| **会签机制** | 多人会签、少数服从多数（V2） |
| **审批操作** | 通过/拒绝/退回/委托 |
| **退回重审** | 审批不通过退回起草人，需填写退回原因 |
| **超时提醒** | 24h 未审批 → 通知 + 邮件；48h 未审批 → 升级至上级 |
| **批量审批** | 勾选多份合同一键通过（V2） |

### 7.3 审批规则配置（示例）

| 合同类型 | 金额阈值 | 审批流程 |
|----------|---------|---------|

## 8 安全与合规设计

### 8.1 数据安全

| 安全措施 | 实现方式 |
|----------|---------|
| **数据加密** | 敏感字段（金额、信用代码）AES-256 加密 |
| **传输安全** | HTTPS + JWT Token 认证 |
| **权限控制** | RBAC 角色权限 + 数据行级权限 |
| **操作审计** | 全链路操作记录，不可删除 |
| **文件安全** | SHA-256 文件哈希校验，防篡改 |
| **数据备份** | 每日自动备份，保留 30 天 |

### 8.2 权限体系

平台采用 **RBAC（基于角色的访问控制）** + **数据行级权限** 双重保障。

| 角色 | 核心职责 | 数据范围 |
|------|---------|---------|
| **起草人** | 创建合同、提交审批 | 本人合同（读写） |
| **部门主管** | 审批本部门合同 | 本部门合同（只读）+ 本人待办 |
| **法务** | AI 审查复核、风险条款处理 | 全公司合同（读写） |
| **财务** | 财务条款审查 | 全公司合同（只读）+ 本人待办 |
| **高管** | 重大合同审批 | 全公司合同（只读）+ 本人待办 |
| **档案管理员** | 合同归档、台账维护 | 全公司合同（读写） |
| **系统管理员** | 用户管理、流程配置、系统监控 | 全公司所有数据（读写） |

### 8.3 合规性保障

- 符合《民法典》合同编法规要求
- 符合《电子签名法》电子签章规范
- 符合《企业内部控制基本规范》流程管控
- 参照 ISO 27001 信息安全管理体系标准
- 审查结论强制引用真实法规条文，禁止编造

### 11.3 典型使用场景

**场景一：采购合同签订**

1. 业务人员从模板库选择"采购合同"模板
2. 选择已有相对方档案，系统自动带入企业名称、信用代码等信息
3. 填写金额、交货周期等关键信息
4. 提交合同，系统自动发起 AI 审查
5. 30 秒内生成五维度审查报告，标记 2 条中风险条款
6. 法务复核后确认，系统按金额（50万）自动路由至标准审批流程
7. 部门主管→法务→财务→高管依次审批
8. 审批通过后申请用印，档案管理员二次确认
9. 上传盖章后扫描件，系统自动计算文件哈希
10. 合同归档至台账，设置到期提醒

**场景二：AI 风险发现**

1. 法务收到一份"服务合同"AI 审查报告
2. 报告显示：合规性 85 分、风险条款 72 分、财务条款 90 分、履约 78 分、异常 88 分
3. 发现"违约责任"条款中未设置赔偿上限，AI 标记为高风险（置信度 92%）
4. 引用《民法典》第 584 条作为法律依据
5. AI 同时提供修改建议："建议增加赔偿上限条款，不超过合同总额的 30%"
6. 法务一键采纳修改建议，修订后重新触发审查
7. 审查风险降至低风险，合同继续流转

## 13 总结

合同审批管理平台是一款以 **AI 智能审查**为核心的全生命周期合同数字化管理解决方案。平台通过四大核心能力——**模板化起草、AI 五维审查、策略化审批、全链路溯源**——帮助企业实现合同管理的规范化、智能化和数据化。

### 核心优势

1. **AI 驱动**：基于 Qwen3.6 本地大模型 + RAG 增强 + 规则引擎三重能力，审查漏判率 <10%，审查耗时 <30 秒
2. **开箱即用**：Docker 一键部署，预制 5 种标准模板、50 条核心法规、6 种预置角色，上线即用
3. **本地化部署**：AI 推理完全本地，无需外网，数据不出域
4. **策略驱动**：审查规则、审批阈值、风险偏好均可配置，适配不同行业和企业规模
5. **全链路可追溯**：从起草到归档，每个操作均有审计记录，文件哈希校验防篡改
6. **合规保障**：审查结论强制引用真实法律条文，内置置信度评估与法务反馈闭环
