# 合同审批管理平台 - 产品需求文档（PRD）

> 版本：V1.0 | 日期：2024-01-15 | 状态：评审中

---

## 一、产品定位与目标

### 1.1 产品定位
全生命周期数字化合同审批管理平台，覆盖"起草→审批→签署→履约→归档"全流程，以 AI 智能审查为核心能力，降低法律风险、提升审批效率、实现数据可查可控。

### 1.2 核心目标

| 目标 | 指标 | 验收标准 |
|------|------|---------|
| **规范审批流程** | 审批时长缩短 30% | 从平均 5 天降至 3.5 天 |
| **严控法律风险** | 高风险条款漏判率 <10% | AI 审查 + 法务复核双保险 |
| **提升起草效率** | 模板使用率 >80% | 标准合同模板覆盖 80%+ 场景 |
| **数据可查可控** | 合同台账完整率 100% | 所有合同可追溯、可统计、可导出 |
| **履约风险预警** | 逾期/异常预警率 >95% | 里程碑节点自动提醒 |

## 二、用户角色与权限

### 2.1 角色定义

| 角色 | 职责 | 核心操作 |
|------|------|---------|
| **起草人** | 业务人员，发起合同起草 | 创建合同、选择模板、上传文件、提交审批 |

## 三、核心业务模块需求

### 3.1 合同起草（MVP）

#### 用户故事
> **作为** 业务人员，**我希望** 能快速起草合同，**以便** 尽快提交审批。

#### 功能清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **空白起草** | 手动填写合同基本信息 + 上传文件 | P0 |
| **模板起草** | 从标准模板库选择模板，自动填充条款 | P0 |
| **历史引用** | 引用已归档合同作为新合同基础 | P1 |
| **草稿保存** | 支持中途保存，后续继续编辑 | P0 |
| **版本管理** | 每次提交生成新版本，保留修改历史 | P0 |
| **文件上传** | 支持 PDF/Word/图片，最大 50MB | P0 |
| **相对方选择** | 从相对方库选择或新建 | P0 |

#### 验收标准
- [ ] 支持 5 种标准模板（采购/销售/劳务/保密/NDA）
- [ ] 模板起草 3 步完成：选模板→填关键信息→提交
- [ ] 草稿自动保存（每 5 分钟），刷新不丢失
- [ ] 文件上传支持拖拽，显示上传进度
- [ ] 相对方支持搜索（名称/信用代码），新建时自动校验信用代码格式

#### 用户故事

#### 功能清单

| **模板分类** | 按合同类型（采购/销售/劳务等）分类管理 | P0 |
| **版本管理** | 模板版本迭代，旧版本标记"已废止" | P0 |
| **发布审批** | 新模板/模板修订需法务负责人审批后生效 | P1 |
| **模板引用** | 起草时强制/推荐模板使用，偏离模板自动高亮 | P0 |
| **模板权限** | 控制谁可创建/修改/发布模板 | P1 |

#### 验收标准
- [ ] 模板支持变量标记：`{金额}`、`{日期}`、`{相对方}`
- [ ] 模板列表显示：名称、类型、版本、状态、最后更新时间
- [ ] 废止模板不可被新建合同引用，已引用合同不受影响
- [ ] 模板修订需法务负责人审批，审批通过后新版本生效

### 3.4 AI 合同审查（MVP）

#### 用户故事
> **作为** 法务专员，**我希望** AI 自动审查合同风险，**以便** 快速识别高风险条款。

#### 功能清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **合规性审查** | 主体资格、经营范围、强制性规定、格式条款 | P0 |
| **风险条款识别** | 违约责任、管辖约定、解除条件、保密条款 | P0 |
| **置信度评估** | 每条审查结论附带置信度（0-100%） | P0 |
| **规则引擎** | 违约金上限、管辖法院、预付款比例等硬规则 | P0 |
| **审查报告** | 结构化报告：风险等级、评分、条款详情、修改建议 | P0 |
| **法务复核** | 确认/修正 AI 审查结论，反馈闭环 | P1 |
| **行业适配** | 支持切换行业规则包（建筑/软件/贸易等） | P2 |
| **修订建议生成** | AI 直接输出修改后条款文本（diff 格式） | P2 |

#### 审查维度与评分

| 维度 | 权重 | 检查项 | 评分标准 |
|------|------|--------|---------|
| **合规性** | 30% | 主体资格、经营范围、强制性规定 | 违反强制性规定 → 直接高风险 |
| **风险条款** | 25% | 违约责任、管辖、解除、保密 | 赔偿无上限 → 高风险 |
| **财务条款** | 20% | 付款条件、价格、税务 | 预付款>50% → 高风险 |
| **履约能力** | 15% | 交付周期、质量标准、验收条件 | 验收条件模糊 → 中风险 |
| **异常检测** | 10% | 矛盾条款、隐藏风险、缺失条款 | 条款矛盾 → 高风险 |

#### 置信度分级

| 置信度 | 处理策略 |
|--------|---------|
| **≥80%** | AI 直接标记风险等级，法务快速复核 |
| **60-79%** | AI 标记"疑似风险"，强制人工复核 |
| **<60%** | AI 标记"不确定"，转人工审查 + 记录争议点 |

#### 验收标准
- [ ] 审查耗时 <30 秒（10 页以内合同）
- [ ] 高风险条款漏判率 <10%（基于 50 份历史合同测试）
- [ ] 每条审查结论附带法律依据（如《民法典》第 584 条）
- [ ] 支持一键导出审查报告（PDF/Word）
- [ ] 法务复核后可标记"AI 误报"/"AI 漏报"，用于模型优化

### 3.5 审批流程（MVP）

#### 用户故事
> **作为** 审批人，**我希望** 快速审批合同，**以便** 不阻塞业务进度。

#### 功能清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **标准审批流程** | 起草→部门主管→法务→财务→高管→用印→归档 | P0 |
| **简易审批流程** | 起草→部门主管→法务→用印（小额合同） | P0 |
| **特殊审批流程** | 起草→部门主管→法务→财务→高管→董事会→用印（重大合同） | P0 |
| **条件分支** | 根据金额/类型自动选择流程 | P0 |
| **审批操作** | 通过/拒绝/退回 | P0 |
| **审批委托** | 将待办转交他人处理 | P1（V1 原型可示意） |
| **会签** | 多人会签、少数服从多数 | P2（V1 不做） |
| **审批意见** | 自由文本 + 常用意见快捷选择 | P0 |
| **待办列表** | 显示待审批合同列表，按紧急程度排序 | P0 |
| **超时提醒** | 24h/48h 未审批，自动提醒 + 升级 | P1 |
| **批量审批** | 勾选多份合同一键通过 | P1 |
| **草稿预审** | 正式提交前内部沟通，支持批注/评论 | P2 |

#### 审批规则配置

| 合同类型 | 金额阈值 | 审批流程 |
|---------|---------|---------|
| **采购合同** | ≤10 万 | 简易流程（部门主管→法务→用印） |
| **采购合同** | 10-100 万 | 标准流程（部门主管→法务→财务→高管→用印） |
| **采购合同** | >100 万 | 特殊流程（+董事会审批） |
| **销售合同** | ≤50 万 | 简易流程 |
| **销售合同** | >50 万 | 标准流程 |
| **劳务合同** | 不限 | 标准流程 |

#### 验收标准
- [ ] 合同提交后自动匹配审批流程
- [ ] 审批人可在待办列表直接查看合同详情 + 审批
- [ ] 退回操作需填写退回原因，合同返回起草人
- [ ] 超时 24h 未审批，系统通知 + 邮件提醒
- [ ] 超时 48h 未审批，自动升级至审批人上级

### 3.7 履约跟踪（V2）

> **V1 范围**：仅状态看板展示「执行中 / 即将到期」合同；里程碑、变更、续签等属 V2。到期提醒在 V1 通过看板 + 消息中心示意。

#### 用户故事
> **作为** 业务人员，**我希望** 跟踪合同履约进度，**以便** 及时发现逾期/异常。

#### 功能清单（V2）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **里程碑管理** | 定义交付/验收/付款分期节点 | V2 |
| **履约状态** | 正常/延期/异常/完成 | V2 |
| **到期提醒** | 到期前 30/15/7 天自动提醒 | V1 看板示意 / V2 全自动 |
| **变更管理** | 合同变更/补充协议流程 | V2 |
| **提前终止** | 提前解除申请 + 审批 | V2 |
| **续签流程** | 到期前自动触发续签流程 | V2 |

#### V1 验收标准（看板）
- [ ] 看板展示执行中、即将到期（≤30 天）、已到期合同
- [ ] 到期合同可跳转台账筛选

#### V2 验收标准（完整履约）
- [ ] 里程碑节点逾期自动标记"延期"状态
- [ ] 到期前 30 天触发续签提醒，推送至起草人 + 部门主管
- [ ] 变更/补充协议需重新走审批流程（可引用原合同条款）

## 八、风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| **AI 审查准确率不足** | 法务不信任 AI，V1 无法上线 AI 功能 | 中 | V1 前用 50 份历史合同验证，准确率<80% 则 AI 作为可选辅助 |
| **审批流程不匹配实际** | 业务方抱怨流程僵化 | 高 | V1 硬编码 3 种流程覆盖 80% 场景，V3 提供可视化设计器 |
| **模板使用率低** | 业务人员习惯自由起草 | 中 | 强制模板使用 + 自由起草需法务额外审核 |
| **相对方数据不全** | 上线后相对方库为空 | 高 | 提供 Excel 批量导入工具，上线前完成历史数据迁移 |
| **移动端审批体验差** | 高管习惯桌面端审批 | 低 | V1 仅提供通知 + 快捷操作，完整审批仍走桌面端 |

|------|------|
