### 🔴 P0 问题（3 项）

#### P0-1：无角色切换能力，所有页面视角固定为「采购部主管」
**影响**：无法验证权限矩阵 — 法务/财务/档案管理员/系统管理员的菜单项、操作按钮完全不同。原型始终以「张三 · 采购部 · 部门主管」视角呈现，导致：
- 法务专员应看到的「AI 审查复核」功能缺失验证
- 档案管理员应无「新建合同」按钮
- 系统管理员的「用户管理」页面完全缺失

**建议**：在顶栏增加角色切换下拉框（张三/法务/财务/管理员），对应显示不同菜单和操作按钮。

#### P0-2：合同生命周期状态转换缺失「已签署」「已终止」状态
**影响**：PRD 定义的完整生命周期为 `草稿→审批中→已通过→用印中→已签署→执行中→已归档`，原型仅有 `草稿/待审批/已通过/已归档` 四种状态，缺少：
- 「已签署」状态：审批+用印完成后的中间状态
- 「执行中」状态：签署后在履约跟踪期间的状态
- 「已终止」状态：提前终止/作废的合同

**建议**：增加状态枚举，合同列表筛选器补充对应选项。

#### P0-3：合同到期/逾期判断仅靠显示，无自动预警触发逻辑
**影响**：看板展示「即将到期 7 份」「已逾期 3 份」为硬编码数字，但：
- 无到期时间计算逻辑
- 无预警阈值配置（30/15/7 天）
- 无「已提醒 N 次」的动态状态
- 无逾期自动升级机制

**建议**：JS 增加日期计算 + 预警状态模拟，展示预警流程节点。

### 🔴 P0 问题（2 项）

#### P0-4：刷新页面后所有状态丢失 — 无 Session 状态保持
**影响**：原型核心演示路径（起草→审批→用印→归档）无法完整验证，因为：
- 合同列表是硬编码数组，filterContracts() 不修改数据源
- 审批操作后仅修改 Toast 文字，不更新表格行状态
- 评审工作台确认进度刷新后归零

**建议**：使用 `localStorage` 保存交互状态，确保审批确认/标记确认后刷新不丢失。

#### P0-5：合同列表无分页 — 150+ 条合同一次性渲染
**影响**：原型演示 10 条数据尚可，但 PRD 要求「支持大数据量下的高效筛选与浏览」，无分页：
- 无分页控件
- 无页码跳转
- 无每页条数选择

**建议**：增加简单前端分页（每页 20 条），含上一页/下一页/页码显示。

### 🔴 P0 问题（1 项）

#### P0-6：状态标签颜色与语义不一致
**影响**：多处存在标签颜色与业务含义冲突，直接影响理解：
- 「保密协议」类型标签使用 `tag-gray`（业务类型→灰色），但「采购」用 `tag-blue`，无统一语义
- 「已归档」状态用 `tag-gray`，但「已通过」state 也用 `tag-gray`（line 2043）→ 状态与状态混淆
- 「普通」紧急程度用 `tag-gray`，但「已归档」也用 `tag-gray` → 紧急程度与状态混淆

**建议**：
- 业务类型：统一用 `tag-blue`
- 状态标签：`tag-green`(成功态)、`tag-yellow`(进行态)、`tag-red`(异常态)、`tag-gray`(终止/归档态)
- 紧急程度：用图标而非颜色标签（🔴紧急/🟡一般/🟢普通）

### 🔴 P0 问题（5 项）

#### P0-7：缺失「用户管理」页面
**PRD 定义角色**：系统管理员需「用户管理、角色权限」，原型无此页面。直接影响：
- 无法演示用户新增/编辑/禁用
- 无法演示角色分配（起草人/审批人/法务/财务/管理员）
- 审批配置页中的「审批人类型: dept_head/role_code」无配置入口

**建议**：增加「用户管理」页面（表格：姓名/部门/角色/状态/操作），放入「系统」导航组。

#### P0-8：缺失「合同详情独立页面」
**当前实现**：合同详情仅为模态框（600px 宽），信息展示受限。PRD 要求：
- 显示完整合同内容（条款原文）
- 显示审批流程进度
- 显示 AI 审查摘要
- 显示附件列表
- 显示操作记录

**建议**：将合同详情提升为独立页面，或模态框扩展至 900px + 3 列布局。

#### P0-9：相对方详情缺失
**PRD 要求**「相对方详情页显示：基本信息 + 历史合同列表 + 履约统计」。原型仅有「详情」按钮但无对应页面/模态框。点击无响应。

**建议**：增加相对方详情模态框（基本信息 + 历史合同子表 + 统计数字）。

#### P0-10：模板管理无创建/编辑模态框
**PRD 要求**「创建模板：上传文件 + 标记可变字段 + 在线编辑」。原型：
- 有「+ 创建模板」按钮但无弹窗
- 有「编辑」按钮但无弹窗
- 有「预览」按钮但无弹窗

**建议**：增加模板创建/编辑/预览模态框，展示模板变量标记功能。

#### P0-11：合同列表/台账无删除操作
**PRD 未明确要求物理删除**，但原型中合同列表操作列仅有「查看」和条件「审批」，无删除/作废入口。归档台账页同样无操作入口。

**建议**：为草稿状态合同增加「删除」（软删除）按钮，已归档合同增加「作废」按钮。
