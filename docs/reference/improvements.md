## 详细改进记录

### 1. AI 审查知识库基座（P0）

**改进内容**：
- 新增“法规知识库基座”章节，包含 50 条核心法规清单示例
- 设计知识库结构：`LawArticle` 模型（法规名称、条款号、原文、摘要、标签、风险场景）
- 实现语义检索器：`KnowledgeBaseRetriever`（Chroma 向量库 + text2vec 嵌入）
- 增强 Prompt 模板：注入【参考法规】上下文，强制 AI 引用真实条款，严禁编造

**验收标准**：
- V1 内置≥50 条核心法规
- AI 审查结论引用法规准确率≥90%
- 幻觉控制：编造率=0
- 检索相关性：Recall@5≥80%

---

### 2. 审批规则配置化（P0）

**改进内容**：
- 新增“审批规则配置化”章节，替代硬编码阈值
- 设计 `approval_thresholds` 表：合同类型 + 金额区间 → 流程模板 ID
- 设计 `approval_node_approvers` 表：流程节点 + 审批人类型（user_id / role_code / dept_head）
- 实现动态审批人解析器：`ApproverResolver`，支持按角色/部门主管动态解析

**验收标准**：
- 金额阈值支持后台动态配置，无需发版
- 审批人支持 role_code / dept_head 解析
- 配置修改后实时生效

---

### 3. 状态看板与到期提醒（P0）

**改进内容**：
- 新增“合同状态看板”API：`GET /api/v1/contracts/dashboard`
- 看板分三栏：执行中、即将到期（≤30 天）、已到期
- 新增“到期提醒配置”API：`POST /api/v1/reminders/expiration`
- 支持自定义提醒天数（30/15/7）、接收人、通知渠道

**验收标准**：
- 看板数据分类准确率 100%
- 到期前 30/15/7 天自动触发提醒
- 支持系统通知 + 邮件 + 飞书/企微多渠道

---

### 4. 数据库企业级预留（P0）

**改进内容**：
- `contracts` 表新增 `org_id` 字段（默认 1），支持 V2 多组织扩展
- `contracts` 表新增 `deleted_at` 字段，支持软删除与误删恢复
- 新增对应索引：`idx_org`、`idx_deleted`

**验收标准**：
- 单组织模式下 `org_id=1` 不影响现有查询
- 软删除合同可通过后台恢复，数据完整

---

### 5. API 接口扩展（P1）

**改进内容**：
- 新增审批规则配置 API：
  - `GET/POST/PUT /api/v1/config/thresholds`

- 新增状态看板 API：`GET /api/v1/contracts/dashboard`
- 新增到期提醒 API：`POST /api/v1/reminders/expiration`

**验收标准**：
- 配置 API 支持 CRUD 操作
- 看板 API 返回结构化数据（执行中/即将到期/已到期）
- 提醒 API 支持自定义配置
